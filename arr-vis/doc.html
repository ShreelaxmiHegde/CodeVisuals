Arrays

Arrays are linear data structures that store elements of the same data type in contiguous memory locations.


  • All elements must be of the same data type.
  • Each element is accessible via its index in constant time.
  • arr[0] — Accessing the element at index 0.
  • arr[0] = 30 — Error if the array is declared as constant or unmodifiable.
  • int arr[4]; — Declares an array of size 4 with uninitialized (garbage) values.
  • int arr[4] = {1, 2, 3, 4}; — Declares and initializes an array with 4 elements.
  • int arr[10] = {1, 2, 3, 4}; — Remaining elements will be initialized to 0.
  • int arr[] = {1, 2, 3, 4}; — Compiler infers array size from values.
  • Note: Accessing out-of-range indices causes undefined behavior or warnings.
  • Arrays occupy contiguous memory blocks.
  • Memory is allocated statically at compile time.
  • Arrays are passed by reference to functions.

Arrays as Pointers in C++

  • Array names are treated as pointers in C++.
  • These two function declarations are equivalent:
    • void func(int arr[])
    • void func(int* arr)
  • cout << arr; — Prints the memory address (pointer).
  • *arr gives arr[0], *(arr + 2) gives arr[2], etc.

⚠️ Beware: Size Issue Inside Functions

When you pass an array to a function, sizeof(arr) only gives the size of the pointer, not the full array, because both point to the same memory. So this won’t work: 

    void printArr(int arr[]) {
        int n = sizeof(arr)/sizeof(int); // Won’t give actual size
        for(int i = 0; i < n; i++) {
            cout << arr[i];
        }
    }

Instead, pass the array size as an argument:

    void printArr(int arr[], int n) {
        for(int i = 0; i < n; i++) {
            cout << arr[i];
        }
    }

📌 Short Notes and Tricks

  • Get size of array (only works in the same scope):
    sizeof(arr) / sizeof(arr[0]) — Returns the number of elements.
  • Input into array:
    Use a loop:
        for(int i = 0; i < n; i++) {
        cin >> arr[i];
    }
  • Output from array:
    Use a loop:
        for(int i = 0; i < n; i++) {
        cout << arr[i] << " ";
    }

🔡 Char Arrays

Char arrays are sequences of characters stored in continuous memory. They end with a special character called the null character ('\0'), which marks the end of the string.


📦 Basics & Initialization

  • char arr[] = {'a', 'b', 'c', 'd', '\0'}; – Manual initialization with null character at the end.
  • All char array rules are similar to regular arrays.
  • cout << arr prints the full string, not just the address like in int arrays.
  • strlen(arr) gives the number of characters before '\0'.
  • arr[3] – Accessing a single character is the same as with int arrays.

✍️ Creation & Output

  • char arr[] = "code"; – Automatically stores: {'c', 'o', 'd', 'e', '\0'}
  • char arr[5] = "code"; – Fits perfectly including the null character.
  • char arr[50] = {'a', 'b', 'd'}; – Rest of the array contains garbage or empty values.
  • arr[2] = 'r'; – You can update characters like normal arrays.

🧾 Input in Char Arrays

  • char word[20]; cin >> word; – Reads a single word (stops at space or newline).
  • cin.getline(arr, size); – Reads full sentences including spaces.
    Optional: cin.getline(arr, size, delimiter);

📦 Dynamic Arrays

Memory allocation using the new keyword

        int size;
        cin >> size;                  // Take array size at runtime
        int* arr = new int[size];     // Dynamically create an array of that size

🧹 Since this memory is created manually, it must be freed manually too using the delete keyword: 

        delete[] arr;   // Correct way to delete a dynamic array

*Always use delete[] (not just delete) when freeing a dynamic array.
